# アイコンデザインメモ

このメモは、Play Store アイコンを書き出す前に MdLite Reader のアプリアイコンの
方向性を定義します。

## 目的

ローカル Markdown ビューアであることが伝わり、洗練され、軽量で、信頼できる印象の
アイコンにします。

このアイコンは次の場所で使える必要があります。

- Android ランチャーの adaptive icon。
- Google Play の 512 x 512 px アプリアイコン。
- Google Play のフィーチャーグラフィックとスクリーンショットの視覚方針。

## ブランド印象

MdLite Reader の印象:

- 速い。
- 静か。
- 実用的。
- 読むことに集中している。
- プライバシーを重視している。
- 装飾過多ではない現代的な見た目。

避けるもの:

- 重い 3D 効果。
- 写真的な細部。
- 特徴のない汎用ドキュメントアイコン。
- 純粋なモノクロのユーティリティ感。
- ランキング、価格、バッジ、宣伝文言。
- Google Play、GitHub、その他の第三者ブランドに似た見た目。

## 中心モチーフ

推奨モチーフ:

```text
Markdown 文書 + 読書ペイン + 控えめなタブ端
```

理由:

- 文書形状はローカルファイルを伝えやすい。
- 読書ペインは編集ではなく閲覧を伝えやすい。
- タブ端は MVP の主要機能とつながる。
- Markdown 記法は軽く示す程度にし、主役にしない。

アイコンを `MD` の文字ロゴ中心にはしません。小さいランチャーサイズでは文字が
崩れやすく、ローカライズ上の懸念もあります。

## 形状

太く、中央に置かれ、輪郭がはっきりしたシンボルを使います。

- adaptive icon の安全領域内に、角丸四角形または文書面を置く。
- シンプルに保てるなら、折り返し角や重なりを使ってよい。
- メイン文書の背後に、控えめなタブ層を 1 つ置いてよい。
- 重要な要素は adaptive icon のマスク端に寄せない。

小サイズのルール:

48 x 48 px でも、ひとつの文書/ビューアのシンボルとして読めること。細かな
Markdown 行は装飾であり、アイコン理解に必須としません。

## 色の方向性

単色感のない、バランスの取れた配色にします。

推奨方向:

- 文書上の記号には深いインク色。
- 文書面には白に近い淡い寒色。
- 新鮮さとプライバシー感のために teal / green 系アクセント。
- 読みやすさとコントラストのために、少量の warm yellow アクセント。

避けるもの:

- 紫または紫青グラデーションが支配的な配色。
- ベージュ、クリーム、砂色、茶色、オレンジ、エスプレッソが支配的な配色。
- ダークブルーまたはスレートが支配的な配色。
- ストア素材の主背景としての純黒または純白。

ドラフトパレット:

```text
Ink: #17212B
Document: #F8FAF7
Teal: #2AA889
Yellow accent: #F2C94C
Soft shadow: #496057
```

最終実装でコントラスト調整はしてよいですが、意図は維持します。

## ディテール量

入れてよいディテール:

- 2 から 4 本の短い Markdown 風の行。
- 見出し線 1 本。
- チェックボックスまたは bullet 1 つ。
- 小さなテーブル/コードの示唆。ただし読める場合のみ。

避けるもの:

- 密なテキスト行。
- 小さすぎる `#`、`*`、コード記号。
- 3 枚以上の重なった紙。
- 強い影を複数入れること。

## Adaptive Icon ルール

Android ランチャーアイコンは adaptive icon のままにします。

- background と foreground を分ける。
- foreground の内容は安全領域内に収める。
- 透明な端の細部に依存しない。
- round icon と standard icon は同じ foreground を使う。

更新対象の現在のリソース:

```text
src/main/res/drawable/ic_launcher_background.xml
src/main/res/drawable/ic_launcher_foreground.xml
src/main/res/mipmap-anydpi-v26/ic_launcher.xml
src/main/res/mipmap-anydpi-v26/ic_launcher_round.xml
src/main/res/mipmap/ic_launcher.xml
src/main/res/mipmap/ic_launcher_round.xml
```

## Play Store アイコンルール

必要な出力:

```text
play-store/icon-512.png
```

要件:

- 512 x 512 px。
- alpha ありの 32-bit PNG。
- 1024 KB 以内。
- 価格、ランキング、ストアバッジ、宣伝文言を入れない。
- ランチャーアイコンと視覚的に一致する。

## フィーチャーグラフィックとの接続

フィーチャーグラフィックは、アイコンを大きく置くのではなく、アイコンの視覚言語を
展開します。

使うもの:

- 同じ文書形状の言語。
- 同じ teal と yellow のアクセント。
- タブと Markdown 内容を含むシンプルな読書シーン。

避けるもの:

- アイコンをそのまま大きく配置して主役にすること。
- フィーチャーグラフィック内に密な UI スクリーンショットを入れること。
- 文字の多いスローガン。

## 受け入れチェックリスト

- [ ] 48 x 48 px で認識できる。
- [ ] 読める文字に依存していない。
- [ ] adaptive icon の foreground が安全領域内に収まっている。
- [ ] 配色が単調な紫、ベージュ、茶/オレンジ、ダークブルーに見えない。
- [ ] ローカル文書を読むアプリであることが伝わる。
- [ ] 編集、クラウド同期、オンラインサービスを示唆しない。
- [ ] Play Store アイコンを 512 x 512 px で書き出せる。
- [ ] フィーチャーグラフィックへ同じ視覚言語を展開できる。
