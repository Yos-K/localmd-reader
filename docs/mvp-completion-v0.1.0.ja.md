# v0.1.0 MVP 完成判定

この文書は MdLite Reader v0.1.0 の完成条件を定義します。

## MVP の目的

MdLite Reader v0.1.0 は、Android でローカル Markdown ファイルを確実に開き、
広告、トラッキング、ログイン、同期、ネットワークアクセスなしで読める状態を
完成とします。

このリリースでは、小さく、速く、信頼できることを重視します。完全な Markdown
プラットフォームである必要はありません。

## 必要なユーザー成果

- アプリ内ファイル選択から `.md` ファイルを開ける。
- アプリ内ファイル選択から `.markdown` ファイルを開ける。
- Android のファイルマネージャから Markdown ファイルを開ける。
- Termux から `mdlite-reader` で 1 つ以上の Markdown ファイルを開ける。
- 見出し、段落、リスト、チェックリスト、引用、コードブロック、インライン
  コード、リンク、テーブル、水平線を含む一般的な Markdown 文書を読める。
- ライトテーマとダークテーマを切り替えられる。
- ピンチ操作で読書用フォントサイズを調整できる。
- 複数ファイルをタブとして開ける。
- アプリ再起動後に開いていたタブを復元できる。
- ローカル履歴から最近開いたファイルを開ける。
- 最近開いたファイル履歴をクリアできる。
- 非対応、サイズ超過、読み取り不可のファイルについて、開けない理由が分かる。
- アプリUIから、個人情報を収集しないことを確認できる。

## 必要なエンジニアリング成果

- `android.permission.INTERNET` を要求しない。
- WebView の JavaScript が無効である。
- WebView のファイルアクセスが無効である。
- Raw HTML は表示前にエスケープされる。
- HTTP / HTTPS リンクのみ外部アプリで開く。
- 最近開いたファイルとタブ復元のメタデータはアプリ専用領域に留まる。
- `./test.sh` が成功する。
- アプリID、署名、権限、ランチャーアイコンのリリースチェックが成功する。
- 本番署名鍵、シークレット、トークン、アカウント情報、プライベートパスを
  コミットしない。

## ストア準備の成果

- アプリアイコンが確定している。
- フィーチャーグラフィックが書き出されている。
- スマホスクリーンショットが 4 枚以上ある。
- 英語の Play Store 掲載文が準備できている。
- 日本語の Play Store 掲載文が準備できている。
- プライバシーポリシー文書が準備できている。
- データセーフティ申告方針が準備できている。
- リリース署名手順が文書化されている。

## リリース前に必要な手動確認

- `/sdcard/Download/mdlite-reader-debug.apk` をインストールする。
- ランチャー名とアイコンを確認する。
- ファイル選択から開けることを確認する。
- Android のファイルマネージャから開けることを確認する。
- Termux から 1 ファイルを開けることを確認する。
- Termux から複数ファイルを開けることを確認する。
- 非対応、サイズ超過、読み取り不可のファイルのダイアログを確認する。
- タブを閉じる動作とタブ復元を確認する。
- ライトテーマとダークテーマを確認する。
- ピンチによるフォントサイズ変更を確認する。
- テーブルの横スクロールを確認する。
- 言語切り替えを確認する。
- 操作バー位置の復元を確認する。
- 最近開いたファイルと履歴クリアを確認する。
- プライバシーダイアログの文言を確認する。

詳細な手順は `docs/manual-device-check-v0.1.0.md` にあります。

## v0.1.0 から明示的に除外するもの

- Markdown 編集。
- 完全な CommonMark 互換。
- Mermaid。
- 数式。
- 脚注。
- リモート画像読み込み。
- 相対画像表示。
- クラウド同期。
- PDF エクスポート。
- HTML エクスポート。
- フォルダ閲覧。
- Git 連携。
- アプリ内購入または Pro 解放。

## 画像表示の判断

画像表示は v0.1.0 から除外します。

理由:

- リモート画像読み込みはネットワーク権限を必要とし、初期版の
  ネットワークなしというプライバシー方針と衝突する。
- 相対画像表示は、document URI とファイル権限の扱いを明確に設計する必要がある。
- 画像が壊れて表示されると、ビューアとしての信頼感が下がる。
- v0.1.0 は、安全なローカル Markdown テキスト閲覧に集中する。

画像対応は、ファイル権限モデルを明確に設計したあと、後続バージョンで再検討します。

## 完成判定

v0.1.0 は次の状態で MVP 完成とします。

- 必要なユーザー成果が実装されている。
- 必要なエンジニアリング成果を満たしている。
- ストア準備用の素材と文言が存在する。
- リリース直前に `./test.sh` が成功する。
- 手動確認項目が確認済み、または明示的に延期されている。
- 延期項目が v0.1.0 のブロッカーではないと文書化されている。
