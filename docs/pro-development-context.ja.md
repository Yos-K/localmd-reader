# Pro 開発コンテキスト

この文書は、クローズドテストのテスター募集が 12 人に到達した後、Pro 機能開発へ
戻るためのコンテキストリセットとして残します。

## 現在のプロダクト状態

LocalMD Reader v0.1.0 は、ローカルファイル向けの軽量 Android Markdown
ビューアです。Free アプリにはすでに次が含まれています。

- `.md` と `.markdown` ファイルを開く。
- Android ファイルマネージャと Termux から開く。
- 制限した WebView による安全なローカル Markdown 表示。
- ライトテーマとダークテーマ。
- ピンチによるフォントサイズ変更。
- タブ、タブ復元、最近開いたファイル。
- 横スクロール可能なテーブル表示。
- 英語と日本語の UI。
- 広告、トラッキング、ログイン、クラウド同期、`INTERNET` 権限なし。

アプリは Google Play のクローズドテスト中です。ストア素材とランチャーアイコンは
更新済みです。このコンテキスト作成時点で、クローズドテストへアップロード済みの
最新ビルドは versionCode 7 です。

## 既存の Pro 基盤

コードベースには、明示的な Free / Pro 境界がすでにあります。

- `FeatureEntitlement`
- `FeatureEntitlements`
- `ViewerFeature`
- `ProFeatureCatalog`
- `ProFeatureDescriptor`
- `ProFeaturePresentation`
- `ProFeaturePresentationItem`

現在のクローズドテスト版の権利状態は Free のままです。Pro 実装では、UI コード内に
生の boolean を散らさず、機能アクセスはドメインオブジェクトを通します。

Markdown レンダラは、フェンスコードの安全な言語名を `language-...` class として
残すところまで実装済みです。これはシンタックスハイライトの土台です。

## Pro 機能の原則

- Pro は読みやすさとパワーユーザー向けの作業効率を高める。
- Pro によって、プライバシー、オフライン利用、広告なし方針を弱めない。
- Pro 機能は local-first を維持する。
- Pro 判定は Markdown を開いて読む中心経路から分離する。
- Pro 権利確認ができない場合でも、Free のビューアとして使える状態を保つ。
- Free は基本的な Markdown 閲覧体験として完結している必要がある。

## 初期 Pro 機能の順序

1. フェンスコードブロックのシンタックスハイライト。
2. 追加の読書テーマ。
3. カスタムジェスチャーショートカット。
4. Mermaid 図の表示。

この順序にする理由は、シンタックスハイライトが既存の言語 class 基盤を活かせて、
ファイルアクセス、保存、UI ナビゲーションへの影響が最も小さいためです。

## 最初の実装単位

最初の開発単位は、ローカルなシンタックスハイライトのパイプラインです。

- レンダラにコードハイライト用のオプションを追加する。
- デフォルトの Free 挙動は、エスケープ済みのプレーンなコードブロックのままにする。
- 小さなローカルハイライト実装を追加する。
- 最初は Java 系キーワードから始める。アプリ本体が Java 実装であり、既存テストも
  Java フェンスコードを使っているため。
- セキュリティを維持する。コード内容は必ずエスケープし、ハイライトによって
  生 HTML が混入しないようにする。
- ページビルダにハイライト用 CSS class を追加する。

最初の実装単位では Play Billing は不要です。実際の購入連携は後回しにし、
ドメイン設定または権利状態に応じたレンダリングモードでゲートできる形にします。

## 後回しにする Pro 作業

最初のハイライト実装が安定するまで、次は後回しにします。

- Play Billing 連携。
- オフライン権利キャッシュ。
- Pro 購入 UI。
- テーマカタログ UI。
- ジェスチャーショートカット設定 UI。
- Mermaid レンダラ選定。

## テストルール

実装は TDD で進めます。

- テスト名から、対象仕様と確認したいことが分かるようにする。
- 関連する境界値をテストする。
- 任意の Markdown テキストを受ける処理では、必要に応じて生成入力による
  プロパティテスト的な安全性確認を追加する。
- Free 挙動と Pro 挙動を両方テストする。
- 安全性テストでは、生 HTML や危険な属性が出力されないことを確認する。

## リリースメモ

Pro 機能のコードは公開リポジトリで開発してよいです。秘密情報、署名鍵、
サービスアカウント JSON、Play Console 認証情報は引き続きリポジトリ外に置きます。
