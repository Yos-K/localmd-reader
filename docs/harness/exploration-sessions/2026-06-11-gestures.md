## proposed-rule（オーナー裁可待ち・S2 分類ゲート）

観測挙動の正典化には意図/欠陥の分類が必要（[domain-knowledge-loop.md](../domain-knowledge-loop.md)）。

| ID | 観測 | intent なら | defect なら |
|----|------|------------|------------|
| PR-1 | `swipe_*` トリガーは直線スワイプで発火せず、シェブロン（往復）のみ認識（P2） | 「直線はスクロールと衝突するため意図的にシェブロン」→ glossary に L2 として正典化＋設定UIの文言確認 | トリガー名・UI文言と実挙動の乖離 → 直線認識の追加 or 名称変更を issue 化 |
| PR-2 | 閉じた正方形・三角形など「閉じた非往復経路」全般が円と判定される（P7） | 「手描きの揺れを広く許容する設計」→ glossary L3 に判定実態を正典化 | GES5 の「別図形が誤一致する」と同型 → 真円度チェック追加を issue 化 |
| PR-3 | 経路認識の最小サイズが px 基準で、発火に必要な物理サイズが端末密度依存（P12） | 「全端末で px 一律が意図」→ glossary に明記 | 高密度端末で誤発火しやすくなる → dp 換算への統一を issue 化 |
