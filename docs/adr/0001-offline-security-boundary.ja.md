# ADR-0001: Readerをアーキテクチャとしてオフラインに保つ

状態: 採用

## 決定事項

`INTERNET` permissionを要求しない。文書Reader WebViewではJavaScriptを無効にする。
広告、追跡、アカウント、リモート設定を使わず、機能はローカルデータだけで動作させる。

## 検討した選択肢

- 解析、クラッシュ報告、リモート素材のために通信を許可する。
- Markdown拡張のためReader WebViewでJavaScriptを有効にする。
- Readerから通信とJavaScriptを排除し、例外的な描画を制約されたローカル部品へ分離する。

## 選択理由

権限とスクリプトが存在しないことをビルド時に監査でき、信頼できないMarkdownの攻撃面を減らし、
接続がなくてもアプリの価値を維持できるため。

## 選択しなかった理由

方針文だけでは偶発的な権限追加を防げない。Readerでのスクリプト実行は文書と実行機能の境界を壊す。
リモートサービスは識別、可用性、依存コストを追加し、プロダクトの性質と合わない。

## 決定を見直す契機

プロダクトの根本的な性質を変更するとオーナーが明示承認した場合、または同じプライバシー境界を
検証可能な形で保つAndroidの分離機構が利用できる場合だけ見直す。
