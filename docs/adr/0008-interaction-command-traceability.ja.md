# ADR-0008: 高リスクな操作コマンドをモデル・コード・テストで追跡する

状態: 採用

## 決定事項

高リスクな操作コマンドを機械可読な契約へ登録し、各状態遷移を一つの実装入口と一つの動作テストへ
結び付ける。実装入口には`interaction-command`を記載し、ファイルとlocatorの実在をローカルpreflightと
必須CIで検査する。探索で欠陥が見つかった操作、追加・変更する操作から段階的に契約対象へ加える。

## 検討した選択肢

- ADRの文章だけを操作の根拠とする。
- 状態モデル内部の整合性だけを検査する。
- 過去の全コマンドへ直ちに契約を要求する。
- 高リスクまたは変更対象のコマンドから段階的に追跡する。

## 選択理由

正しい状態グラフだけではAndroidのタッチ処理が振る舞いを実装したと証明できない。反対に、モデル遷移や
動作テストを持たない実装マーカーは、未文書化かつ無防備な操作を示す。locator単位の追跡ならfitnessで
エミュレータを動かさず差分を早期検出でき、結び付けたmedium testで実行時の振る舞いをCI検証できる。

## 選択しなかった理由

文章だけではドリフトを拒否できない。モデルだけの検査では、スクロール容器の存在を確認しても縦スワイプで
下部操作へ到達できることを証明できなかった。全履歴を一度に登録すると、弱い仮テストを正当化し、重要経路を
改善する代わりに埋没させる。

## 決定を見直す契機

全操作コマンドに意味のある動作テストが揃った場合、ソースマーカーなしで対応を導出できる意味解析を導入した
場合、または計測した保守コストが防止できた回帰損失を上回る場合に段階導入の範囲を見直す。
