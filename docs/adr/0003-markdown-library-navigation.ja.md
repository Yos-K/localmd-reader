# ADR-0003: Markdownライブラリを継続的なメニュー内ツリーにする

状態: 採用

## 決定事項

ProのMarkdownライブラリは、通常のファイル操作と分けて、スクロール可能なハンバーガーメニュー内の
折り畳みツリーとして表示する。操作行には目次と同じ展開・折り畳み記号を付ける。読み込み済みなら、
操作行を押すたびに選択したルート、現在地、絞り込みを失わず開閉する。未読み込みの場合だけAndroidの
フォルダー選択を開く。文書を開くと閲覧のためメニューを閉じてもよいが、再度開いたときもツリーの状態を
維持し、次の関連文書を少ない操作で選べるようにする。

## 検討した選択肢

- 文書ごとにAndroidのファイル選択を開く。
- ライブラリを一回限りの選択ダイアログとして表示する。
- 専用の全画面プロジェクトブラウザーを作る。
- 既存メニュー内に継続的なツリーを置く。

## 選択理由

ライブラリの価値は一つのファイルを選ぶ別手段ではなく、関連ファイル間を繰り返し移動することにある。
メニューはボタンと端スワイプから到達でき、文書画面を別モードへ置き換えずナビゲーション面として使えるため。

## 選択しなかった理由

ファイル選択の繰り返しはプロジェクト文脈を失う。ダイアログは一ファイルで利用経路を終了し、通常の
ファイル選択と重複する。全画面ブラウザーは、現状の規模では不要な最上位モードを増やす。

## 決定を見直す契機

プロジェクト操作がメニューの規模を超える場合、アクセシビリティ試験でツリー操作の信頼性が不足する場合、
または全画面方式が文書閲覧を阻害せず操作数を減らせると実証された場合に見直す。
