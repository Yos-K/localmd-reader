# ADR-0017: Freeは一つのファイル選択、Proは継続的なライブラリを使う

状態: 採用

## 決定事項

Freeでローカルファイルを選ぶ入口は、Android標準のファイル選択だけとする。一回限りの
「フォルダーから選ぶ」は表示しない。ProではADR-0003に従い、選択したルートとメニュー内の
展開ツリーを文書選択後も保持するMarkdownライブラリを提供する。

## 検討した選択肢

- Freeに通常のファイル選択と一回限りのフォルダー選択を両方残す。
- FreeにもProと同じ継続的なMarkdownライブラリを提供する。
- FreeとProの両方からフォルダー単位の移動をなくす。
- Freeは一つのファイル選択に統一し、継続的なライブラリ移動をProに限定する。

## 選択理由

Freeの一回限りの経路は、通常のファイル選択にフォルダー選択の一手を加える一方、移動状態を
保持しない。Proのライブラリはプロジェクトのルートとツリーを保持し、関連文書を繰り返し
切り替えられる点で異なる価値を持つ。Freeを完整なReaderに保ちつつ、Proを意図的な不便の
解消ではなく反復操作の利便性向上として成立させられる。

## 選択しなかった理由

Freeに一時的な入口を二つ置くと、同じ目的の操作を理解しにくくする。Freeにも継続的な
ライブラリを提供すると既存の利便性境界が消える。両方から削除すると、文書群を扱う有用な
操作を失う。採用案では各入口の振る舞いを明確に区別できる。

## 決定を見直す契機

Android標準のファイル選択で単一ファイルを実用的に選べなくなった場合、継続的な
ライブラリが基本的な閲覧に不可欠だという根拠が得られた場合、またはFree/Proの製品モデルを
大きく変更する場合に見直す。
