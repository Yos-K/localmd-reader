# ADR-0004: Android Storage Access Frameworkを使う

状態: 採用

## 決定事項

ファイルとプロジェクトルートはAndroid Storage Access Frameworkから開く。許可された永続read権限を保持し、
文書をcontent URIで識別する。記憶した権限が無効になった場合はファイル選択へ戻って回復する。

## 検討した選択肢

- 広範囲または全ファイルのストレージ権限を要求する。
- 生のファイルシステムパスを直接移動する。
- 独自のストレージプロバイダーまたはファイルマネージャーを作る。
- システム文書プロバイダーと限定URI権限を使う。

## 選択理由

ユーザーがアクセス対象を制御でき、ローカルと各種プロバイダーで動作し、広い権限なしに
プロジェクトへの継続アクセスを実現できるため。

## 選択しなかった理由

広い権限は最小権限とPlayの期待に反する。生パスはプロバイダーや現代のAndroidストレージで安定しない。
独自マネージャーはプラットフォーム機能を重複させ、プロバイダー境界も安全には越えられない。

## 決定を見直す契機

AndroidがSAFを置き換える場合、永続権限が利用できなくなる場合、または同じプロバイダー範囲を
より単純な利用経路で扱える限定APIが提供された場合に見直す。
