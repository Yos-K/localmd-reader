# ADR-0010: 文書一覧ダイアログを一つの排他的セッションとしてモデル化する

状態: 採用

## 決定事項

最近開いたファイル、ピン留め、フォルダー内ファイルのダイアログを、単一の
`DocumentListDialogState`が持つ排他的variantとして表現する。項目選択と副操作は明示的な
`DocumentListCommand`へ変換する。専用の`DocumentListDialogController`がセッションを所有して
Androidの効果へ変換し、`MainActivity`は要求を委譲するだけにする。

## 検討した選択肢

- ダイアログ種別ごとのbooleanと表示データを`MainActivity`に保持する。
- 現在のタイトルやボタン文言から一覧種別を推測する。
- 共通モデルを持たず、一覧種別ごとにAndroid listenerを分ける。
- 一つの排他的セッションモデルと一つのAndroidアダプタを使う。

## 選択理由

独立したbooleanは矛盾状態を許し、共通のAndroid callbackがpresentationフィールドからユーザーの意図を
復元する原因になっていた。明示的variantなら元の一覧と有効な結果が一意になり、全域な選択操作により、
遅延したcallbackや範囲外indexもモデル化されたno-opになる。ダイアログ制御を分離することで、
`MainActivity`の責務も減らせる。

## 選択しなかった理由

booleanを増やしても不正な組み合わせは残る。タイトルと文言はローカライズされた描画値であり、正しい状態の
根拠にはできない。listenerだけを分ける案は分岐を減らせても選択規則を重複させ、ハーネスとテストが検証する
単一のモデルを提供できない。

## 決定を見直す契機

文書一覧を常設画面またはパネルへ移す場合、一覧固有のコマンドが共通の選択契約を持たなくなった場合、
またはAndroidのナビゲーション部品が同等のalways-validかつテスト可能な保証でセッションを所有できる場合に、
ダイアログ表現を見直す。
