# ADR-0012: 番兵状態を使わず見出し移動を導出する

状態: 採用

## 決定事項

見出し移動の結果をalways-validな`HeadingNavigation`のvariantである`Unavailable`と
`Destination`で表現する。次・前への移動コマンドが実行されるたびに、現在の文書の目次と
スクロール位置から移動先を導出する。`MainActivity`には独立した可変の見出しindexを保持しない。

## 検討した選択肢

- `activeHeadingShortcutIndex`を保持し、`-1`を現在の見出しなしとして扱う。
- 現在の見出しをoptionalとして保持し、文書変更ごとにリセットする。
- Androidの各ジェスチャーcallback内で移動先を直接推測する。
- 文書の見出しとスクロール位置から型付けされた移動値を一つ導出する。

## 選択理由

ユーザーが読んでいる位置については、表示中のWebView位置がすでに正である。別の可変indexは、
スクロール、文書を開く操作、タブ切り替えにより古くなる。`Unavailable`は見出しのない文書と
文書外の見出し選択を明示し、`Destination`は次・前への移動が必ず範囲内となり、両端で循環することを保証する。

## 選択しなかった理由

`-1`の番兵値は無効な算術を許し、境界判定を`MainActivity`へ分散させる。optionalな見出しを保持しても、
スクロールや文書ライフサイクルとの同期が必要になる。ジェスチャーごとの推測は同じ移動規則を重複させ、
目次から再利用できない。

## 決定を見直す契機

rendererから見出しの可視性を正確に通知できる場合、スクロール位置とは独立してユーザーが選んだ見出しを
保持する必要が生じた場合、または移動履歴が製品要件になった場合に見直す。
