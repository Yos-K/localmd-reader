# ADR-0007: ADR未確認の機能追加・修正コミットを拒否する

状態: 採用

## 決定事項

重要な設計判断を英日ADRに残し、すべての `feat:` / `fix:` コミットへ `ADR-Review:` トレーラを
1つだけ要求する。宣言、参照先、対訳文書、必須節をローカルフック、preflight、必須CI fitnessで検証する。

## 検討した選択肢

- 設計判断をPR本文だけに残す。
- ADRの指針だけを書き、自動化しない。
- マージを止めない助言的なADRレポートを実行する。
- 機能追加・修正コミットごとに確認宣言を強制する。

## 選択理由

リポジトリ再作成や後続実装により、以前のUX・アーキテクチャ文脈が見えなくなることがある。
コミット単位の宣言なら変更と同時に確認でき、必須ゲートならエージェントや人間の経路による黙った省略を防げる。

## 選択しなかった理由

PR本文は一時的な実装文脈で、後から参照不能または見落とされ得る。強制のない指針では、実際に
継続的なメニューツリーの決定が反転した。助言だけの検査は受け入れ経路を閉じない。

## 決定を見直す契機

誤検知による実測コストが防止できる回帰損失を上回る場合、コミットがレビュー単位でなくなる場合、
または監査性を弱めずトレーラを置き換える意味的な影響検査が利用可能になった場合に見直す。
