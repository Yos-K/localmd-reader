# ADR-0005: ローカルMermaid描画をReaderから分離する

状態: 採用

## 決定事項

同梱したMermaid互換レンダラーを、制約された別の描画部品で非同期実行する。メインReader WebViewでは
JavaScriptを無効のままにし、ネットワークからレンダラーを取得せず、生成SVGを無害化する。
描画に失敗した場合はソースを読める状態で残す。

## 検討した選択肢

- 文書ReaderでMermaid JavaScriptを実行する。
- Mermaidまたは描画サービスをリモートから取得する。
- 完全な図形エンジンをJavaまたはWASMですぐに自作する。
- 図を省く、または同梱ローカルレンダラーを分離して使う。

## 選択理由

図は技術Markdownに大きな価値がある。分離により信頼できない文書との境界を保ち、非同期処理で
閲覧応答性を守り、同梱によりオフラインで決定的に動作できるため。

## 選択しなかった理由

Readerのスクリプト実行はADR-0001を弱める。リモート描画は内容を外部へ渡し通信を要求する。
新規ネイティブエンジンは互換性と保守コストが高く、図を省くと主要な利用例を失う。

## 決定を見直す契機

互換性、ライセンス、サイズ、テーマ、安全性、性能を満たす小さく成熟したネイティブまたはWASM実装が、
分離した同梱実装より明確に優れる場合に見直す。
