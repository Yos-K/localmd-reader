# ADR-0019: ピン留め文書を開いているタブから独立させる

状態: 採用

## 決定事項

ピン留め文書は、開いているタブが所有する属性ではなく、保存済みMarkdownファイルへの
永続的なブックマークとしてモデル化する。Proのファイルタブは長押しでピン留めを切り替え、
文書がピン留め中であればピンアイコンとローカライズ済みのアクセシビリティ説明を表示する。
メニューと長押しは同じピン留め・解除の振る舞いを使い、変更時はタブ表示とメニュー操作を
まとめて更新する。

タブを閉じても文書のピン留めは解除しない。ピン留め一覧では、選択したファイルを開く、
そのファイルだけ解除する、すべてのピン留めを明示的に解除する操作を提供する。
ウェルカムタブと一時下書きタブはピン留め文書モデルへ入れない。

## 検討した選択肢

- ピン留めを、タブを閉じると消える開いているタブの属性とする。
- メニューからのみピン留めできるようにする。
- ピン留め一覧では全解除だけを提供する。
- 永続ブックマークを独立してモデル化し、その状態をタブへ投影する。

## 選択理由

利用者は、ファイルを閉じた後やアプリ再起動後にも戻れるようにピン留めする。そのため、
ブックマークはタブより長く存在する。長押しは対象となるタブで直接操作でき、アイコンは
操作結果を可視化する。変更処理の共有により、タブのアイコンとメニュー操作の不一致を防ぐ。
個別解除により、不要な一件のために他のブックマークまで失わずに済む。

## 選択しなかった理由

タブ所有のピン留めは、タブを閉じたときに再訪手段を失う。メニューのみの操作は頻繁な操作を
タブから隠す。全解除のみでは一件を消すために有効なブックマークも破棄させる。変更経路を
分けた過去の実装では、メニュー操作後にピンアイコンが残る不整合が発生した。

## 決定を見直す契機

Androidによりアクセシブルな標準ブックマーク操作が提供されたとき、長押しが重要な
アクセシビリティ操作と競合するとき、またはピン留め文書がローカルURIを超えた識別子を持つ
同期コレクションになるときに見直す。
