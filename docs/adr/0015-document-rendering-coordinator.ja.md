# ADR-0015: プラットフォーム非依存コーディネータによる描画状態の所有

状態: 採用

## 決定事項

`DocumentRenderingCoordinator`を`DocumentRenderingSession`の唯一の所有者とする。
文書を開く、Mermaid完了、テーマ再描画、タブを閉じる操作はこのコーディネータを経由させる。
描画、バックグラウンドジョブ投入、表示中タブ更新は小さな`Output`ポートとして公開し、Androidのpresentationアダプタが実装する。

`MainActivity`は命令と現在の描画設定を渡してよいが、描画セッションを直接保持・置換しない。

## 検討した選択肢

- セッションと結果適用手順を`MainActivity`に残す。
- 同じ手順をAndroid依存コントローラへ移す。
- 各呼び出し元へ次セッションを返し、効果を個別に適用させる。
- Outputポートを持つプラットフォーム非依存コーディネータを使う。

## 選択理由

描画遷移では、ジョブ投入や画面更新より先に次セッションを権威ある状態にする必要がある。
純粋なコーディネータならこの順序を一つのテスト可能な振る舞いにでき、入口ごとの適用漏れを防げる。
OutputポートによりAndroid、WebView、具象レンダラを状態所有者から分離し、高速なJVMテストで手順を検証できる。

## 選択しなかった理由

`MainActivity`に残すと可変状態の所有と手順が集中したままになる。Androidコントローラへの移動だけでは結合が残り、
順序検証にRobolectricが必要になる。各呼び出し元へセッションを返す方法は部分適用を許し、正しさが呼び出し側の規律に依存する。

## 決定を見直す契機

描画がプロセス単位サービスへ移る場合、複数ウィンドウで一つの描画セッションを共有する場合、またはActivityの生存期間を越える
バックグラウンド処理のため永続的なセッション所有者が必要になった場合に見直す。
