## 現在のプロダクト状態

LocalMD Reader v0.1.0 は、ローカルファイル向けの軽量 Android Markdown
ビューアです。Free アプリにはすでに次が含まれています。

- `.md` と `.markdown` ファイルを開く。
- Android ファイルマネージャと Termux から開く。
- 制限した WebView による安全なローカル Markdown 表示。
- フェンスコードブロックのシンタックスハイライト。
- 基本的なローカル Mermaid 図表示。
- ライトテーマとダークテーマ。
- ピンチによるフォントサイズ変更。
- ダブルタップショートカット。
- タブ、タブ復元、最近開いたファイル。
- 横スクロール可能なテーブル表示。
- 英語と日本語の UI。
- 広告、トラッキング、ログイン、クラウド同期、`INTERNET` 権限なし。

このスナップショット作成時点では、アプリは Google Play のクローズドテスト中で、
ストア素材とランチャーアイコンは更新済み、クローズドテストへアップロード済みの
ビルドは versionCode 7 でした。この段落を現在のリリース状態として扱わないでください。

## 既存の Pro 基盤

コードベースには、明示的な Free / Pro 境界がすでにあります。

- `FeatureEntitlement`
- `FeatureEntitlements`
- `ViewerFeature`
- `ProFeatureCatalog`
- `ProFeatureDescriptor`
- `ProFeaturePresentation`
- `ProFeaturePresentationItem`

このスナップショット作成時点では、クローズドテスト版の権利状態は Free のままでした。
現在の実装でも、UI コード内に生の boolean を散らさず、機能アクセスはドメインオブジェクトを通します。

Markdown レンダラは、フェンスコードの安全な言語名を `language-...` class として
残すところまで実装済みです。これはシンタックスハイライトの土台です。
Markdown パーサ移行基準は
[`markdown-parser-migration-criteria.ja.md`](markdown-parser-migration-criteria.ja.md)
で定義します。

## Pro 機能の原則

- Pro は読みやすさとパワーユーザー向けの作業効率を高める。
- Free は購入前にコアの読みやすさを体験できる完成品にする。
- Pro は頻繁に使う操作をより速く、柔軟に、快適にする。
- Pro によって、プライバシー、オフライン利用、広告なし方針を弱めない。
- Pro 機能は local-first を維持する。
- Pro 判定は Markdown を開いて読む中心経路から分離する。
- Pro 権利確認ができない場合でも、Free のビューアとして使える状態を保つ。
- Free は基本的な Markdown 閲覧体験として完結している必要がある。

## 初期 Pro 機能の順序

1. 追加の読書テーマ。
2. ダブルタップ以外の高度なジェスチャーショートカット。
3. 目次と見出しジャンプ。
4. 最近開いたファイルの拡張とピン留め。

この順序にする理由は、コアの読書品質を Free に置いたまま、頻繁に使う人へ
速度と快適さの明確なアップグレード理由を提供できるためです。

## 最初の実装単位

最初の開発単位では、Free の Markdown リーダーとしての完成度を保ちながら、
Pro を快適化機能へ寄せます。

- コードハイライトと基本的な Mermaid 表示は Free で有効にする。
- ダブルタップショートカット設定は Free で有効にする。
- 高度なジェスチャー、追加テーマ、長文ドキュメント向け便利機能は Pro でゲートする。
- セキュリティを維持する。コード内容は必ずエスケープし、ハイライトによって
  生 HTML が混入しないようにする。

## テストルール

実装は TDD で進めます。

- 実装およびリファクタリングでは、先にテストを修正または追加する。
  production code を先に変更し、その後で実装に合わせてテストを追加する進め方はしない。
- 新しい振る舞いを追加するときは、最小の失敗する仕様テストを先に書く。
- リファクタリングでは、変えてはいけない振る舞いの characterization test を先に確認または
  追加し、テストが green のまま構造を変える。
- バグ修正では、先にバグを再現する失敗する regression test を書き、その後で修正する。
- `kawasima/savanna-maven-plugin` が扱うテストスメルを避ける。
- テスト名から、対象仕様と確認したいことが分かるようにする。
- 関連する境界値をテストする。
- 任意の Markdown テキストを受ける処理では、必要に応じて生成入力による
  プロパティテスト的な安全性確認を追加する。
- Free 挙動と Pro 挙動を両方テストする。
- 安全性テストでは、生 HTML や危険な属性が出力されないことを確認する。
- テスト本文に条件分岐 (`if`, `while`, `switch`) を置かない。
- テストで `Thread.sleep()` を使わない。
- ビルドコマンドやプロジェクトスクリプトでテストをスキップしない。
- 各テストファイルに独自 assert ヘルパーを作らない。
  `io.github.yosk.mdlite.testing.TestAssertions` を使う。
- 各 assertion には、期待する仕様が分かるメッセージを付ける。

## リリースメモ

Pro 機能のコードは公開リポジトリで開発してよいです。秘密情報、署名鍵、
サービスアカウント JSON、Play Console 認証情報は引き続きリポジトリ外に置きます。

## 第三者アセットのルール

第三者のコードまたはアセットを同梱する場合は、正確なバージョン、上流 URL、
パッケージ URL、ライセンス、同梱ファイルを `THIRD_PARTY_NOTICES.md` に記録します。

Mermaid 表示では、現在 `mermaid@11.15.0` を MIT ライセンスで同梱しています。
ライセンス本文は `docs/third-party/mermaid-11.15.0-MIT.txt` に保持し、
`./test.sh` で通知ファイルが残っていることを確認します。

他アプリや Web サイトからアイコン、スクリーンショット、ストア画像、フォント、
JavaScript をコピーしません。再配布が明示的に許可されている場合だけ利用します。
