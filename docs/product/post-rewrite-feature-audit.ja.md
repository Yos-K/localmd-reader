# 公開履歴再構築後の機能監査

## 目的

公開用履歴の再構築前に存在した利用者向け機能が、現在のソースと回帰テストに残っているかを
確認する。コミットIDは履歴間で比較できないため、再構築前の最終ツリー、現在の実装、テストを
機能単位で照合した。

## 結果

| 機能 | 状態 | 根拠・対応 |
|---|---|---|
| メニュー領域の縦スクロール | 維持 | `SwipeMenuScrollView`と`MenuPanelScrollabilityTest`が存在する |
| Markdownライブラリの継続的なツリー導線 | 再実装済み | ADR-0017と現在のメニュー内ツリー実装で確認した |
| Pro機能説明の完全な日英切替 | 再実装済み | ADR-0018と`ProFeatureCatalogTest`で確認した |
| タブ長押しによるピン留め | 欠落・復元中 | `TabPinningDecision`と操作回帰テストを追加した |
| ピン留めタブのアイコン | 欠落・復元中 | テーマ色に追従する`ic_push_pin_18`を追加した |
| ピン留め一覧からの個別解除 | 欠落・復元中 | 開く／個別解除を選ぶ状態とコマンドを追加した |
| ピン変更後のタブ・メニュー同期 | 欠落・復元中 | すべての変更経路を共通更新処理へ集約した |
| ジェスチャーアクションの一意性 | 欠落 | 現行`put`は同じトリガーだけを置換し、同じアクションを他のジェスチャーに残す |
| 登録済みアクションの割当先表示 | 欠落 | 再構築前の割当ラベル型とテストが現行に存在しない |
| 一時タブを保存済みタブへ置換 | 欠落 | 現行保存処理は保存先を追加で開き、一時タブを置換しない |
| クリップボード由来のMarkdown太字描画 | 欠落 | 現行インライン描画はコード・リンク・画像のみを解釈する |
| HTML/MarkdownコードブロックのRaw/Preview切替 | 欠落 | 現行レンダラーとページCSSに切替UIが存在しない |
| コードフェンスメタデータ対応 | 欠落 | Raw/Preview機能とともに再導入が必要 |

## 復元順序

1. ピン留め操作を完了し、ローカルAPKで確認する。
2. 一時タブの保存置換を復元する。
3. Markdown太字描画を復元する。
4. ジェスチャーの一意性と割当先表示を復元する。
5. Raw/Preview切替とフェンスメタデータを一つの描画変更として復元する。

各復元は、再構築前コードの単純なコピーではなく、現在の所有モデルとADRに適合する失敗テストを
先に追加してから実装する。
