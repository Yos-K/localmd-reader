# 使い方

このガイドは LocalMD Reader v0.1.1 の使い方を説明します。

## ファイルを開く

アプリ内から Markdown を開く:

1. `メニュー` をタップする。
2. `ファイルを開く` をタップする。
3. `.md` または `.markdown` ファイルを選択する。

Android から Markdown を開く:

1. ファイルマネージャを開く。
2. `.md` または `.markdown` ファイルを選択する。
3. Android が開くアプリを確認したら LocalMD Reader を選択する。

他アプリから共有された Markdown を開く:

1. 他アプリ内で `.md` または `.markdown` の添付ファイルを開く。
2. LocalMD Reader を選択する。
3. 文書はタブとして開かれる。以前に開いたタブも、読み取り可能であれば残る。

Pro が有効な場合は、`メニュー` -> `Markdownライブラリ` から選択済みの
プロジェクトフォルダーを閲覧できる。入力欄へファイル名またはフォルダー名を入力すると
現在の階層を絞り込み、入力を消すとその階層の全項目を復元する。ライブラリはメニュー操作の
直下にツリーとして残り、文書を開いたあと再びメニューを開くと同じ階層から選択を再開できる。
操作行の右端にある矢印を押すとツリーを折り畳み、もう一度押すと同じ階層と絞り込みのまま開ける。
Android 11以降では、ダウンロードフォルダー自体をライブラリの起点として選択できない。
ダウンロード内のサブフォルダーを選ぶか、新しく作成する。

## Termux から開く

LocalMD Reader は Termux から Markdown ファイルを直接開けます。

```sh
mdlite-reader README.md
```

複数ファイルをタブとして開く:

```sh
mdlite-reader README.md docs/usage.ja.md
```

Termux コマンドは、Termux 側で Markdown 本文を読み取り、LocalMD Reader に渡します。
これにより、Android の広いストレージ権限を要求せずに Termux ホーム配下の
ファイルを開けます。

詳しいセットアップメモは `docs/termux-command.ja.md` にあります。

## クリップボードから作成

`メニュー` -> `クリップボードから作成` で、クリップボードのテキストを一時的な
Markdown 文書として開けます。

- 開ける項目が 1 件だけの場合は、そのまま開く。
- 開ける項目が複数ある場合は、ダイアログで 1 件以上を選び、`選択した項目を開く`
  をタップする。
- 以前にアプリ内で開いたクリップボード項目は、最大 10 件のアプリ内ローカル履歴
  として残り、あとから再度開ける。
- 一時的な Markdown 文書は自動保存されない。保存する場合は `名前を付けて保存` で
  ファイル名を選んで保存する。

Android の制約により、LocalMD Reader は OS やキーボードが持つクリップボード履歴
全体を読み取れません。アプリに表示される履歴は、LocalMD Reader 内で過去に開いた
クリップボード内容だけです。

## タブ

- 別のファイルを開くとタブが追加される。
- タブをタップすると表示中の文書が切り替わる。
- 閉じる操作でそのタブを閉じる。
- 最後の文書タブを閉じるとウェルカム画面に戻る。
- アプリ再起動後、読み取り可能な文書のタブは復元される。

## 読書用の操作

- 文書上でピンチするとフォントサイズを変更できる。
- `メニュー` -> `文書内検索` から、現在の文書内の文字を検索できる。
  検索バーは文書の上に表示されるため、閲覧領域を大きく隠さずに前後の一致箇所へ
  移動できる。
- `メニュー` -> `設定` -> `テーマ` からライトテーマとダークテーマを切り替える。
- `メニュー` -> `設定` -> `操作バーを下に移動` または `操作バーを上に移動` で
  操作部品の位置を変更する。
- `メニュー` -> `設定` -> `Switch to English` または `日本語に切り替え` で
  アプリ内の表示言語を切り替える。

## 最近開いたファイル

- `メニュー` -> `最近開いたファイル` から最近開いた文書を開ける。
- 最近開いたファイルは最大 5 件。
- 最近開いたファイルのダイアログで `履歴をクリア` を選ぶとローカル履歴を削除する。
- Pro が有効な場合、`メニュー` -> `現在のファイルをピン留め` で表示中のファイルを
  ピン留め一覧に残せる。
- Pro が有効な場合、`メニュー` -> `現在のファイルのピン留めを解除` で表示中の
  ファイルをピン留め一覧から外せる。
- `メニュー` -> `ピン留めしたファイル` からピン留めした文書を開ける。

## Free と Pro 機能

Free ビルドには、ローカル Markdown 表示、コードハイライト、基本的な Mermaid 図、
タブ、タブ復元、最近開いたファイル、クリップボードから作成する一時 Markdown 文書、
ライト/ダークテーマ、フォントサイズ変更、ダブルタップショートカット、言語切り替え、
広告やネットワーク権限のないローカル利用が含まれます。

`メニュー` -> `設定` -> `Pro機能` から、現在のビルドで利用可能またはロック中の
高度な機能を確認できます。

Pro 機能は、よく使う読み方をより速く快適にするためのものです。追加テーマ、
ダブルタップ以外の高度なジェスチャー、目次、テーブル表示強化、最近開いたファイルの
拡張などを中心にします。Free の最近開いたファイルは 5 件まで、Pro では 20 件まで
拡張し、ピン留めしたファイルも使えます。Pro では横に広いテーブルをスクロールしても、
見出し行と先頭列が見えるようになります。Pro では、ジェスチャーショートカットに
次の見出し/前の見出しへ移動する操作も割り当てられます。Pro では、ローカル文書
セット向けの安全な相対 Markdown リンクをアンカーとして残します。Pro では、
ローカル文書セット内の安全な相対 Markdown 画像も表示します。`file:` や
`javascript:` などの危険なスキームはテキストのまま表示します。

Pro が有効な場合は、`メニュー` -> `目次` を開くと現在の文書の見出し一覧が
メニュー内に展開されます。見出しを選ぶと、その位置へジャンプできます。

## プライバシー

`メニュー` -> `設定` -> `プライバシー` からアプリ内のプライバシー説明を確認できます。

LocalMD Reader v0.1.1 には、広告、解析 SDK、ログイン、同期、クラッシュ送信、
ネットワーク権限はありません。

## 制限

LocalMD Reader v0.1.1 はビューアであり、エディタではありません。

v0.1.1 では非対応:

- Markdown 編集。
- 完全な CommonMark 互換。
- 数式。
- 脚注。
- リモート画像読み込み。
- クラウド同期。

## ファイルサイズ制限

10 MB を超えるファイルは開けません。スマートフォン上での応答性を保ち、非常に大きな
文書を予期せずメモリに読み込むことを避けるためです。
